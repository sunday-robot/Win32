# 説明
.netプログラムからWin32APIを呼び出すための各種定義集

# クラス構成
|クラス名|説明|
|---|---|
|Win32Api.Win32|各種関数、定数などを定義したもの。詳細は後述|
|Win32Api.Util|ユーティリティ。<br/>本来はこのようなものは作りたくなかったが、GetLastErrorでエラーの種類を取得し、例外として投げるものだけは面倒すぎるので作成した。|

# Win32クラスのソースファイルの構成
このクラスは長大なものになるので、管理しやすいように、以下のようにソースファイルを分割している。

|ソースファイル名|説明|
|---|---|
|Win32.Constant.cs&nbsp;&nbsp;&nbsp;&nbsp;|定数定義|
|Win32.Struct.cs|構造体定義|
|Win32.Delegate.cs|コールバックをdelegateで定義|
|Win32.Function.cs|関数のdllimport宣言|

# 保守方針
必要になったら各種定義を逐次追加する。<br/>
Utilへの追加は原則行わない。

# ルール
追加する際は、何も考えずに辞書順になるように並べる。<br/>
DLL名(kernel.dllなど)でグルーピングすることも考えたが、
おそらく後になってグルーピングのルール見直しなどしたくなってしまうと思われたので、
そうならないようにこうした。

# 名前空間の名前についての説明
名前空間は、Win32Apiではなく、Win32としたかったが、
クラス名と同じでは呼び出し側が扱いにくいことに気づいたので、Win32Apiという少し長いものにした。
名前空間がWin32では以下のような問題が生じる。
```C#
using Win32;
:
Win32.FunctionName();
// ↑コンパイラは、Win32をクラス名ではなく、名前空間としてしかみなさないため、
// コンパイルエラーになってしまう。
```
